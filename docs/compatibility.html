Built for what public APIs allow.

The current Mac app is useful now, but true app-to-any-output routing across all sources is a deeper audio-backend problem.

Works now

  • Real Core Audio device discovery
  • Default input and output switching
  • System volume, mute, and balance where supported
  • Saved route preferences, setups, profiles, and user-added apps

Experimental

  • Per-app process-tap routing on macOS 14.2+
  • Group Play fan-out through multiple output renderers
  • Live per-route meters and 10-band EQ on routed audio
  • HAL virtual input bridge for mixer software

Requires backend

Reliable SoundSource-style routing for every app, app-as-input devices, and tightly synced multi-speaker playback may require a signed driver, virtual audio device, or service.

What each status means.

AudioRouter is designed to keep limited features visible without making the app feel broken.

Working

The backend confirms it can act

Examples include supported system volume, default device changes, saved setups, and source gain state.

Saved Only

The route is stored for retry

The selected source or output is not currently routeable, so AudioRouter preserves the preference and retries later.

Experimental

The route is attempted live

Process taps and fan-out routes may work for some apps and device formats, then fall back cleanly when macOS blocks them.

Requires Backend

A future driver or service is needed

AudioRouter can show and save the workflow today, but cannot promise complete routing until the backend exists.
