The macOS app is the production focus today. It uses public Core Audio APIs first, then labels experimental process-tap routing honestly when a deeper backend is required.

What works today on macOS.

These are the features AudioRouter can provide with public macOS APIs and the current experimental route engine.

Devices

Real input and output discovery

List connected Bluetooth, built-in, USB, HDMI, AirPlay, aggregate, and virtual devices with sample rate and support flags.

Controls

System output and input control

Switch default devices and adjust volume, mute, and balance when the selected hardware exposes those controls.

Routes

Source-to-output workflow

Choose Spotify, Music, Chrome, or user-added apps, then assign outputs, route gain, setup profiles, and retry behavior visually.

Studio

Group Play and EQ surfaces

Create speaker groups, tune per-route gain, save EQ presets, and see when a route is live, saved only, or backend-required.

Simple routing flow

  1. Open AudioRouter from the menu bar.
  2. Pick a source app in Routes.
  3. Select an output device or Group Play target.
  4. Approve macOS audio capture when prompted.
  5. Use Command-= or Command-- to adjust the selected source by 1%.

Important routing note

Some app-to-device routes are experimental because macOS does not expose every SoundSource-style routing feature through public APIs alone. AudioRouter saves the route and shows the real status instead of pretending unsupported routing is live.
