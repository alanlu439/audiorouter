Windows compatibility is underway.

AudioRouter now has a native Windows 11 path in the repository. The backend is being tested on Windows virtual machines before any public Windows download is published.

01

Windows Core Audio backend

MMDevice endpoint discovery, endpoint volume and mute, session discovery, and per-session volume controls are separated from the macOS app.

02

VM smoke testing

A Windows runner compiles and runs a backend smoke test so audio-device code is validated on a real Windows environment.

03

No Windows download yet

The Windows UI shell is still a scaffold. A Windows ZIP or MSIX will be released only after the VM build and app shell are green.

Built for what macOS allows.

AudioRouter is clear about what is live today, what is experimental, and what requires a deeper audio backend.

Works now

  • Real Core Audio device discovery
  • Default input and output switching
  • System volume, mute, and balance where supported
  • Saved route preferences and setup profiles

Experimental

  • Per-app process-tap routing on macOS 14.2+
  • Group Play fan-out through multiple output renderers
  • Live per-route meters and 10-band EQ
  • HAL virtual input driver for mixer software

Always improving

The app keeps unsupported states visible instead of pretending everything is active. Each release tightens routing reliability and setup flow.
