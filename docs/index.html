Built for fast control

A visual routing desk for your Mac.

01

Quick Router

Pick a source app, choose an output target, adjust gain, mute, retry, and test routes from the menu bar.

02

Real Device Control

Lists connected Bluetooth, built-in, USB, HDMI, AirPlay, and aggregate devices with default output switching.

03

Live Route Attempts

Uses public Core Audio process taps where macOS exposes them, with saved routes when the backend cannot go live.

04

Reliability Center

Shows route health, permission state, output readiness, app detection, retry actions, and exact failure reasons.

05

EQ And Gain

10-band EQ, saved presets, 1% gain steps, and selected-route shortcuts with Command = and Command -.

06

Group Play

Create visual output groups, test speakers, set per-speaker volume, and route one app to multiple outputs where supported.

Public API first

What works now, and what is experimental.

Live today

  • Real Core Audio device discovery
  • Default input and output switching
  • System volume, mute, and balance where supported
  • Running route-app detection
  • Saved route preferences and setup profiles

Experimental

  • Per-app process-tap routing on macOS 14.2+
  • Group Play fan-out through multiple output renderers
  • Live per-route meters and 10-band EQ
  • HAL virtual input driver for mixer software
  • Spotify keep-playing recovery for AirPods pauses

Download and run

Get the latest AudioRouter build.

The public build is distributed as a ZIP from GitHub Releases. It includes the app and a download manual.

Download AudioRouter
  1. Open the ZIP and read DOWNLOAD_AND_USE.md.
  2. Move AudioRouter.app to Applications.
  3. Control-click the app and choose Open.
  4. Approve macOS audio and automation prompts when needed.

The current public ZIP may be unnotarized. Keep Gatekeeper enabled and use the documented Control-click Open flow.
